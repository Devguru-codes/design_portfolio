AI / Voice UX Case Study

Building an AI-Powered Voice Call Centre from Scratch

Multi-Agent Voice Platform with Real-Time STT, LLM Routing, and TTS

Opportunity Type: Disruptive

This represents a Disruptive Opportunity. Traditional call centres rely on human agents and rigid IVR trees. This project replaces the entire stack with an AI-native voice pipeline: real-time speech recognition, LLM-powered reasoning with tool execution, and neural text-to-speech, all orchestrated as a multi-agent swarm that hot-swaps personas during a live call.

Executive Summary

Enterprise call centres are expensive. A mid-size company spends $6-12 per minute on human agents, with 40% of call time wasted on routing and hold music. Existing solutions like Retell AI, Bland AI, and Vapi charge $0.10-0.25 per minute and lock you into proprietary platforms.

This project is a fully open-source alternative. It features a real-time voice pipeline streaming PCM audio over WebSockets, with sub-second latency from microphone input to spoken AI response. The system uses Faster Whisper for speech-to-text, Groq's Llama-3 for LLM reasoning, and Edge TTS for neural voice synthesis.

The key innovation is the multi-agent swarm architecture: a single call is handled by specialized agents (Receptionist, Sales, Support, Scheduling) that seamlessly transfer context to each other mid-conversation. Each agent has its own persona, voice, tool permissions, and prompt-engineered system instructions. The platform includes a full CRM, calendar booking, email dispatch, RAG knowledge base, and post-call analytics dashboard.

Problem Statement

Small and mid-size businesses cannot afford dedicated call centre teams. When they do set up phone support, callers face frustrating IVR menus, long hold times, and agents who lack context. Existing AI voice platforms are expensive, closed-source, and don't offer the flexibility to customize agent behavior, integrate with internal tools, or self-host for data privacy.

Background Context

Market Landscape: The global AI call centre market is projected to reach $4.1 billion by 2027. Key players (Retell AI, Vapi, Bland AI) offer hosted APIs at $0.10-0.25/min. However, enterprises with compliance requirements (healthcare, finance) need self-hosted solutions. Open-source alternatives are fragmented and lack production-grade orchestration.

Technical Context: Three converging technologies make this feasible: (1) Faster Whisper delivers real-time STT on consumer GPUs, (2) Groq's LPU hardware enables sub-200ms LLM inference, and (3) Edge TTS provides free, high-quality neural voices. The missing piece was a production-grade orchestration layer that ties them together with tool calling and agent handoffs.

The architecture was designed around three core principles: sub-second latency, seamless agent handoffs, and extensible tool integration.

Design Process & User Research

User Segmentation

We identified two user groups: the business admin configuring the system, and the end customer making a call.

  • Business Admin: Needs a simple dashboard to configure agent prompts, upload knowledge base documents, and view call analytics. Technical literacy varies widely.
  • End Customer (Caller): Expects a human-like conversation. Zero tolerance for latency, robotic voices, or being asked to repeat themselves. They should never feel "stuck" with the wrong agent.

Primary User Persona: Meera (Business Admin)

Profile: 34-year-old operations manager at a 50-person SaaS startup. She currently handles support calls herself because they can't afford a dedicated team.

Goal: Set up an AI phone system that handles tier-1 support and appointment booking so she can focus on operations.

Pain Point: Tried Vapi but the per-minute cost was unsustainable at their call volume. Tried building with raw APIs but couldn't get agent handoffs to work reliably.

Design Strategy & Solutions

Multi-Agent Swarm Architecture

The core design innovation. Instead of a single monolithic agent, we defined four specialized personas (Receptionist, Sales, Support, Scheduling), each with its own system prompt, voice, and tool permissions. A transfer_agent tool allows any agent to hand off the conversation mid-call. The ConversationManager strips tool-call artifacts from the history during transfers to prevent the new agent from re-executing old actions.

Prompt Engineering for Voice

Voice AI prompts differ fundamentally from chat prompts. We engineered each agent's system prompt with: (1) explicit formatting rules (no markdown, no emojis), (2) gibberish detection to handle STT noise, (3) strict routing decision trees for the Receptionist, and (4) tool execution verification for the Scheduling Agent.

Key Technical Challenges & Solutions

  • Agent Self-Transfer Loop: The LLM sometimes tried to transfer to the same agent it was already running as (e.g., Support transferring to Support). We added a runtime guard that detects self-transfers and injects a system message telling the agent to help directly.
  • STT Hallucinations: Whisper transcribes background noise as "thank you", "bye", or "mm-hmm". We added a blocklist filter and VAD (Voice Activity Detection) pre-processing to suppress phantom transcripts.
  • ChatGPT Misrecognition: Domain-specific terms like "ChatGPT" were transcribed as "charging" or "chad GPT". We solved this with Whisper's initial_prompt parameter to bias the model toward expected vocabulary.
  • Sentence Splitting: A 0.8s silence threshold was too aggressive, splitting sentences mid-pause. We increased it to 1.5s after user testing.

Metrics & KPIs

North Star Metric: Call Resolution Rate
The primary indicator of success is the percentage of inbound calls fully resolved by the AI without human escalation. Our target is 70%+ for tier-1 support and scheduling queries.

Success Metrics

  • End-to-End Latency: Time from user finishing speaking to AI starting its response (target: under 1.5 seconds).
  • Transfer Accuracy: Percentage of calls routed to the correct specialist agent on the first attempt.
  • Tool Execution Success: Rate of successful CRM updates, calendar bookings, and email dispatches during calls.
  • Customer Satisfaction (CSAT): Post-call rating of the AI interaction quality.
  • Cost per Call: Total infrastructure cost per resolved call vs. human agent baseline ($6-12/call).
